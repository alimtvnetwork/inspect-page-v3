# Inspect Page v2.4 — Picker Overlay Redesign + Polish

v2.3.0 shipped Inspect Mode (A1–A14) and end-to-end Stripe billing.
v2.4 focuses on the **picker overlay UI** that was assumed-but-never-built
during v2.3, plus small post-launch polish driven by what we learn from
the Phase 6 manual launch. Driven by `next` — one phase per turn.

## Goals

1. Build the on-page picker overlay so element selection has a visible
   chip + tooltip + action icon (closes the deferred v2.3 UX call).
2. Tighten the billing flow with telemetry + better error states.
3. Carry over lessons from production launch (hardening, copy, a11y).

## Phases

### B1 — Picker chip overlay (HTML + styles)
- Inject a self-contained shadow-DOM overlay from `extension-src/picker/`.
- Floating chip near the hovered element shows the canonical selector
  (`button.relative` style) + element role + size badge.
- Light/dark theme parity using the same CSS variables as the panel.
- Pure presentational; selection logic stays in `picker.ts`.

### B2 — Chip action icon
- Add a small "✕" / "lock" / "open inspector" action icon to the chip.
- Click "✕" cancels the current hover lock without exiting picker mode.
- Click "lock" pins the current selection so the user can move the
  cursor without losing the target.
- Click "open inspector" jumps to the Inspect tab pre-anchored to that
  element (reuses A8 anchor + B3 distance guides).

### B3 — Distance guides
- Dashed lines + px badges between the locked selection and the
  currently hovered element (top / right / bottom / left).
- Inset / overlap detection (negative distance shown in red).
- Reuses `colorMath` distance helper where applicable.

### B4 — Picker keyboard support
- `Esc` exits picker mode.
- `↑` walks to parent element, `↓` to first child, `←` / `→` to
  previous / next sibling.
- `Enter` triggers the same action as the primary chip icon.
- All bindings respect `aria-keyshortcuts` and the WCAG bypass-block rule.

### B5 — Billing telemetry + admin log
- Plugin records `webhook_received` events (type + uid + created_at) in
  a small ring buffer (last 100) so admins can debug Stripe sync.
- New WP admin Billing → "Recent webhook events" table.
- Extension records `billing_checkout_clicked` / `billing_portal_clicked`
  via the existing telemetry channel (no PII).

### B6 — Pricing-page polish from launch feedback
- Tighten `[inspect_page_pricing]` styling once we have the live
  marketing page (probably extract inline styles into a stylesheet,
  add `aria-pressed` state to CTAs).
- Localise all strings into the existing `inspect-page` text-domain.

### B7 — Tests, docs, packaging
- Unit tests for the new picker overlay (jsdom + shadow-root assertions).
- Update `docs/QA-CHECKLIST.md` with AC-PICK-1…N rows.
- CHANGELOG entry, version bump to 2.4.0, rebuild zips.

## Out of scope for v2.4
- Multi-currency Stripe pricing (defer until we see geo demand).
- Team / org licenses (today the license is per-WP-user).
- A picker overlay that survives full-page navigation (single-tab only).
