# 17 — File formats produced by the extension

## A. Full-page bundle (`.zip`)

Filename: see `07-file-naming.md`. Compression DEFLATE level 6.

```
inspect-page-fullpage-{domain}-{timestamp}.zip
├── page.html         UTF-8, ends with single LF
├── styles.css        UTF-8, source-ordered, /* === <source> === */ headers
├── scripts.js        UTF-8, source-ordered, /* === <source> === */ headers
├── screenshot.png    Full-page stitched image (PNG by default)
├── manifest.json     ExportMeta (see schema below)
└── README.txt        3-line human note: what this is + how to feed to an LLM
```

### `manifest.json` (ExportMeta)

```ts
type ExportMeta = {
  schemaVersion: 1;
  kind: 'fullPage';
  url: string;                  // location.href at capture time
  title: string;                // document.title
  capturedAtIso: string;        // ISO 8601 UTC
  viewportCssPx: { w: number; h: number };
  pageCssPx:     { w: number; h: number };
  devicePixelRatio: number;
  userAgent: string;
  counts: {
    inlineStyles: number;
    linkedStylesheets: number;
    unreachableStylesheets: number;
    inlineScripts: number;
    linkedScripts: number;
    unreachableScripts: number;
    captureFrames: number;
  };
  extensionVersion: string;     // from chrome.runtime.getManifest().version
};
```

### Source-marker format (in `styles.css` and `scripts.js`)
```
/* === <inline #3 at line 412> === */
...content...
/* === <https://cdn.example.com/app.css> === */
...content...
/* === <unreachable: https://cdn.example.com/blocked.css> === */
```

### `page.html` rules
- Doctype preserved if present, else `<!DOCTYPE html>` prepended.
- `<base href="{location.href}">` injected as first child of `<head>` so relative URLs in the snapshot resolve when opened standalone.
- All `<script>` and `<link rel="stylesheet">` tags retained (not stripped) so the LLM sees the original references; the merged copies in `styles.css`/`scripts.js` are a convenience.
- `<meta charset="utf-8">` ensured.

## B. Element export (`.md`)

Filename: see `07-file-naming.md`. UTF-8, ends with single LF. Template:

```md
# Element export — {tag}{#id}{.classes}

## Source
- URL: {url}
- Captured: {iso}
- Selector path: {selector}
- Page title: {title}
- Viewport: {vw}×{vh} CSS px @ DPR {dpr}

## outerHTML
```html
{outerHtml}
```

## Matched CSS rules
```css
/* from: <stylesheet href or "inline #N"> */
{rule}

/* from: ... */
{rule}
```

## Computed styles (diff vs default)
```css
{prop}: {value};
...
```

## Screenshot — in page context
![context](data:image/png;base64,{base64Context})

## Screenshot — isolated
![isolated](data:image/png;base64,{base64Isolated})

---
_Generated by Inspect Page v{version}._
```

### Caps
- Each Base64 image MUST be ≤ `MD_IMAGE_MAX_BYTES` (2 MiB raw). If exceeded, downscale to fit (see `20-performance-budgets.md`).
- Total `.md` file MUST be ≤ `MD_FILE_MAX_BYTES` (10 MiB). If exceeded, drop the **isolated** screenshot first, then computed styles, and append a `> Truncated: {reason}` note.

## C. README.txt (inside zip)

Exactly:
```
This bundle was produced by the Inspect Page Chrome extension.
Feed the four files to your LLM together with your restyle prompt.
manifest.json contains capture metadata; screenshot.png is the full page.
```

## D. Determinism

Given identical DOM, identical viewport, identical settings, two runs MUST produce byte-identical `page.html`, `styles.css`, `scripts.js`, and `README.txt`. `manifest.json` differs only in `capturedAtIso`. `screenshot.png` may differ only in pixels affected by animations.
