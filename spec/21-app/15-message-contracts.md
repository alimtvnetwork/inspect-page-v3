# 15 — Message contracts (wire protocol)

All cross-context communication uses `chrome.runtime.sendMessage` / `chrome.tabs.sendMessage` with a single discriminated-union envelope. Raw strings are forbidden.

## Envelope

```ts
type Envelope<K extends MessageKind, P, R> = {
  kind: K;
  requestId: string;   // uuid v4, generated by sender
  payload: P;
  // response shape (delivered via sendResponse):
  // { ok: true; data: R } | { ok: false; error: WireError }
};

type WireError = {
  code: ErrorCode;     // see 09-error-handling.md
  message: string;     // human-readable, English
  detail?: string;     // optional stack/context (never PII)
};
```

## MessageKind enum

| Kind | Sender → Receiver | Purpose |
|---|---|---|
| `RunFullPageExport` | Popup/Panel → SW | Start full-page export pipeline. |
| `RunElementExport` | CS → SW | Picker selected element; ship artifacts. |
| `EnterPickerMode` | Popup/Panel → CS | Activate picker overlay. |
| `ExitPickerMode` | Popup/Panel → CS | Cancel picker. |
| `MountFloatingPanel` | Popup → CS | Inject the in-page panel. |
| `CollectPageArtifacts` | SW → CS | Ask CS for HTML/CSS/JS payload. |
| `BeginScrollCapture` | SW → CS | CS prepares page (hide sticky, scroll to y). |
| `RestoreAfterCapture` | SW → CS | CS restores scroll + sticky styles. |
| `CaptureViewport` | SW → self | Internal: call `chrome.tabs.captureVisibleTab`. |
| `OffscreenAddFrame` | SW → Offscreen | Paste a frame at (x,y) on the canvas. |
| `OffscreenStitchFinish` | SW → Offscreen | Return final PNG/JPEG blob. |
| `OffscreenRenderIsolated` | SW → Offscreen | Render cloned element to PNG. |
| `StatusUpdate` | SW → Popup/Panel | Push `PanelStatus` + message. |
| `GetSettings` / `SetSettings` | any → SW | Read/write `chrome.storage.local`. |
| `Ping` | any → any | Health check; receivers must reply within `PING_TIMEOUT_MS`. |

## Payload / response shapes

### RunFullPageExport
```ts
payload: { tabId: number; settings: Settings }
response: { bundleFilename: string; downloadId: number }
```

### RunElementExport
```ts
payload: {
  tabId: number;
  selectorPath: string;     // CSS path from <html>
  rect: DomRect;            // bounding rect in CSS px
  outerHtml: string;
  matchedCss: string;       // joined with /* from: */ headers
  computedDiff: Record<string,string>;
  isolatedHtml: string;     // self-contained doc for iframe render
}
response: { mdFilename: string; downloadId: number }
```

### CollectPageArtifacts
```ts
payload: { tabId: number }
response: {
  html: string;
  css: string;
  js:  string;
  meta: ExportMeta;         // see 17-file-formats.md
}
```

### BeginScrollCapture
```ts
payload: { y: number; viewportHeight: number; settleMs: number }
response: { actualY: number; dpr: number }
```

### CaptureViewport (SW internal)
```ts
payload: { windowId: number; format: 'png'|'jpeg'; quality?: number }
response: { dataUrl: string }   // strictly < CAPTURE_MAX_BYTES
```

### OffscreenAddFrame
```ts
payload: { dataUrl: string; xPx: number; yPx: number }
response: { framesPlaced: number }
```

### OffscreenStitchFinish
```ts
payload: { format: 'png'|'jpeg'; quality?: number }
response: { blobUrl: string; widthPx: number; heightPx: number; bytes: number }
```

### OffscreenRenderIsolated
```ts
payload: { html: string; widthPx: number; heightPx: number }
response: { dataUrl: string }
```

### StatusUpdate
```ts
payload: { status: PanelStatus; message?: string; progress?: { done:number; total:number } }
response: void
```

### GetSettings / SetSettings
```ts
GetSettings: payload: {} ; response: Settings
SetSettings: payload: Partial<Settings>; response: Settings
```

## Sender × Receiver matrix

```text
                Popup  Panel  CS  SW  Offscreen
RunFullPageExport  →     →            ●
RunElementExport               →      ●
EnterPickerMode    →     →     ●
ExitPickerMode     →     →     ●
MountFloatingPanel →           ●
CollectPageArtifacts           ●      ←
BeginScrollCapture             ●      ←
RestoreAfterCapture            ●      ←
OffscreenAddFrame                     →     ●
OffscreenStitchFinish                 →     ●
OffscreenRenderIsolated               →     ●
StatusUpdate       ●     ●            ←
GetSettings/Set    ↔     ↔     ↔      ●
Ping               ↔     ↔     ↔      ↔     ↔
```
`→` send, `←` receive, `●` handler lives here, `↔` either direction.

## Timeouts

| Kind | Timeout (ms) | On timeout |
|---|---|---|
| `Ping` | `PING_TIMEOUT_MS` (1 000) | Mark receiver unavailable. |
| `CollectPageArtifacts` | 30 000 | Abort export with `E_COLLECT_TIMEOUT`. |
| `BeginScrollCapture` | 5 000 per step | Abort with `E_SCROLL_TIMEOUT`. |
| `CaptureViewport` | 3 000 per frame | Retry once, then `E_CAPTURE_FAILED`. |
| `OffscreenAddFrame` | 2 000 | `E_OFFSCREEN_BUSY`. |
| `OffscreenStitchFinish` | 15 000 | `E_STITCH_FAILED`. |
| `OffscreenRenderIsolated` | 10 000 | `E_ISOLATED_FAILED`. |
| `RunFullPageExport` overall | 120 000 | `E_EXPORT_TIMEOUT`. |
| `RunElementExport` overall | 30 000 | `E_EXPORT_TIMEOUT`. |

All timeout constants live in `shared/constants.ts` (see `20-performance-budgets.md`). No magic numbers in code.

## Versioning
Envelope MUST carry `kind` only — no `version` field in v1. If the protocol changes, bump the extension `version` in `manifest.json`; the SW refuses messages from older content scripts via a `Ping` handshake checking `chrome.runtime.getManifest().version`.
